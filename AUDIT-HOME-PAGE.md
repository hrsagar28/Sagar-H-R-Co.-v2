# Home page audit — `pages/Home.tsx`

**Date:** 2026-05-16
**Scope:** Everything `<Home />` mounts directly, plus the shared chrome that always renders alongside it (Navbar, WhatsApp float, cookie banner, fonts, CSP headers).
**Method:** Static review of source + ESLint output + manual reasoning across UI, UX, accessibility (WCAG 2.2 AA), performance / Core Web Vitals, SEO, security, code quality, and conversion path.

This document is the **plan**. The next prompt will turn each finding into a code change.

---

## Legend

- **Severity:** P0 (blocker), P1 (ship-stopper if it shipped tomorrow), P2 (real bug or measurable regression), P3 (polish / nice-to-have).
- **Effort:** XS (≤ 15 min), S (≤ 1 hr), M (≤ half day), L (multi-day).

## 0. Executive summary

The home page is structurally sound and the React 19 / Compiler upgrade landed cleanly. The biggest opportunities are concentrated in four areas:

1. **Rules-of-React violations** in the hero / nav / float chrome that silently disable the Compiler on those exact files. Fixing these is the highest-leverage code-quality work because it actually changes runtime behaviour (memoization stops being a no-op).
2. **Conversion above the fold is brand-first, not value-first.** The hero asserts identity ("Audit. Taxation. Advisory.") but never tells a prospect _what kinds of clients we serve_, _why pick us_, or _what happens when they click_. The Marquee even duplicates the services rail two scrolls later.
3. **Hero LCP/CLS strategy is fragile.** A hand-rolled DOM-poll preload loop in `index.tsx` watches `<span>` text content to decide when to remove the preload paint. Any copy change to the hero silently breaks the gate.
4. **A few specific lint warnings are real regressions** — `WhatsAppFloat` derives state through an effect, `CookieConsent` reads `handleDecline` before it's declared, `HorizontalScroll` re-renders the entire card rail on every scroll frame.

Total findings: **47** (P0: 0 · P1: 6 · P2: 19 · P3: 22).
